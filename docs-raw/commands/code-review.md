# Code Review Command

未コミットの変更に対する包括的なセキュリティおよび品質レビュー：

1. 変更されたファイルの取得：git diff --name-only HEAD
2. 変更された各ファイルについて、以下をチェック：

**セキュリティ問題（重要）：**

- ハードコードされた認証情報、APIキー、トークン
- SQLインジェクションの脆弱性
- XSSの脆弱性
- 入力バリデーションの欠如
- 不安全な依存関係
- パストラバーサルのリスク

**コード品質（高）：**

- 関数が50行を超えている
- ファイルが800行を超えている
- ネストの深さが4層を超えている
- エラーハンドリングの欠如
- console.logステートメント
- TODO/FIXMEコメント
- 公開APIにおけるJSDocの欠如

**ベストプラクティス（中）：**

- ミューテーションパターン（不変性を使用すべき）
- コードやコメント内での絵文字の使用
- 新規コードに対するテストの欠如
- アクセシビリティ（a11y）の問題

3. 以下の内容を含むレポートを作成：

- 重要度：致命的、高、中、低
- ファイルの場所と行番号
- 問題の説明
- 修正案

4. 致命的または優先度の高い問題が発見された場合はコミットをブロックする

セキュリティ上の脆弱性があるコードは決して承認しないこと！
