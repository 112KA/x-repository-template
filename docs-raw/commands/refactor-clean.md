# Refactor Clean Command

テスト検証を通じて、安全にデッドコードを特定し削除します。

1. **デッドコード分析ツールの実行：**

- knip：未使用の exports とファイルの特定
- depcheck：未使用の依存関係の特定
- ts-prune：未使用の TypeScript exports の特定

2. **`.reports/dead-code-analysis.md` に完全なレポートを生成**
3. **発見事項を重要度別に分類：**

- **安全**：テストファイル、未使用のユーティリティ
- **注意**：API ルート、コンポーネント
- **危険**：設定ファイル、主要なエントリポイント

4. **安全な削除のみを提案**
5. **各削除の実行前：**

- フルテストスイートの実行
- テスト合格の検証
- 変更の適用
- テストの再実行
- テスト失敗時のロールバック

6. **クリーンアップ済み項目のサマリーを表示**

テストを実行する前にコードを削除することは決してありません！
