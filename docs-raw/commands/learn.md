# /learn - 再利用可能なパターンの抽出

現在のセッションを分析し、スキルとして保存する価値のあるパターンを抽出します。

## トリガー

セッション中の任意の時点で、些細ではない（重要度の高い）問題が解決された際に `/learn` を実行してください。

## 抽出対象

以下の内容を探してください：

1. **エラー解決パターン**

- どのようなエラーが発生したか？
- 根本的な原因は何だったか？
- 何によって修正されたか？
- 同様のエラーに再利用できるか？

2. **デバッグ手法**

- 自明ではないデバッグ手順
- 効果的なツールの組み合わせ
- 診断パターン

3. **ワークアラウンド（回避策）**

- ライブラリ特有の挙動（癖）
- API の制限
- 特定バージョン向けの修正

4. **プロジェクト固有のパターン**

- 発見されたコードベースの慣例
- 決定されたアーキテクチャ上の判断
- 統合（インテグレーション）パターン

## 出力形式

`~/.claude/skills/learned/[pattern-name].md` にスキルファイルを作成します：

```markdown
# [説明的なパターン名]

**抽出日:** [日付]
**コンテキスト:** [このパターンが適用される状況の短い説明]

## 問題

[このパターンが解決する問題 - 具体的に記述]

## 解決策

[パターン / 手法 / 回避策]

## 例

[該当する場合、コード例を記載]

## 使用のタイミング

[トリガー条件 - 何をきっかけにこのスキルを呼び出すべきか]

```

## プロセス

1. セッション内から抽出可能なパターンをレビューする
2. 最も価値があり、再利用可能な知見を特定する
3. スキルファイルのドラフトを作成する
4. 保存前にユーザーに確認を求める
5. `~/.claude/skills/learned/` に保存する

## 注意事項

- 些細な修正（タイポ、単純な構文エラーなど）は抽出しないでください
- 一時的な問題（特定の API のダウンタイムなど）は抽出しないでください
- 将来のセッションで時間を節約できるパターンに集中してください
- スキルの焦点を絞ってください（1つのスキルにつき1つのパターン）
