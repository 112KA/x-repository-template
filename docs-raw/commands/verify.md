# Verify Command

現在のコードベースの状態に対して包括的な検証を実行します。

## 説明

以下の正確な順序で検証を実行してください：

1. **ビルドチェック**

- このプロジェクトのビルドコマンドを実行
- 失敗した場合はエラーを報告して停止

2. **型チェック**

- TypeScript/型チェッカーを実行
- ファイル名:行番号を含むすべてのエラーを報告

3. **Lintチェック**

- リンターを実行
- 警告とエラーを報告

4. **テストスイート**

- すべてのテストを実行
- パス/失敗の数を報告
- カバレッジのパーセンテージを報告

5. **Gitステータス**

- 未コミットの変更を表示
- 最後のコミット以降に修正されたファイルを表示

## 出力

簡潔な検証レポートを生成してください：

```
検証：[パス/失敗]

ビルド：    [OK/失敗]
型：        [OK/X 個のエラー]
Lint：      [OK/X 個の問題]
テスト：    [X/Y パス、Z% カバレッジ]
秘密情報：  [OK/X 個発見]
ログ：      [OK/X 個の console.logs]

PR作成の準備：[はい/いいえ]

```

重大な問題がある場合は、それらをリストアップし、修正案を提示してください。

## 引数

$ARGUMENTS は以下の通りです：

- `quick` - ビルド + 型チェックのみ
- `full` - すべてのチェック（デフォルト）
- `pre-commit` - コミットに関連するチェック
- `pre-pr` - フルチェック + セキュリティスキャン
