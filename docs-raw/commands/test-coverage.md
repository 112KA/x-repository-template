# Test Coverage Command

テストカバレッジを分析し、不足しているテストを生成します。

1. **カバレッジ付きでテストを実行**：`npm test --coverage` または `pnpm test --coverage`
2. **カバレッジレポートの分析**（`coverage/coverage-summary.json`）
3. **カバレッジ率が閾値の 80% 未満であるファイルを特定**
4. **カバレッジが不足している各ファイルに対して：**

- テストされていないコードパスを分析
- 関数に対するユニットテストを生成
- API に対する結合テストを生成
- クリティカルなフローに対する E2E テストを生成

5. **新しいテストが合格することを確認**
6. **実施前後のカバレッジ指標を表示**
7. **プロジェクト全体のカバレッジが 80% 以上になるように確保**

**重点項目：**

- 正常系のシナリオ
- エラーハンドリング
- 境界値ケース（null、undefined、空の値）
- 境界条件
